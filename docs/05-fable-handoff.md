## 0. TL;DR — Fable, 이것만 하면 된다

**무한 원탭 아케이드 게임 "PULSE"를 HTML5로 만들어라.**
회전하는 바늘을 정확한 타이밍에 탭 → 맞히면 점수·콤보·속도↑, 빗나가면 게임오버,
**광고 보고 부활**해서 무한히 이어간다. 강한 이펙트 + iPhone 햅틱 + 깔끔·심플.
그리고 **웹으로 배포**해서 iPhone에서 플레이되는 링크를 만들어라.

전체 배경은 `CLAUDE.md`와 `docs/01~04`에 있다. 스펙은 `docs/02`, 기술은 `docs/03`, 배포는 `docs/04`.

## 1. 세션 시작 프롬프트 (복붙용)

Fable 세션을 열고 아래를 그대로 입력하면 된다:

```
이 저장소의 CLAUDE.md와 docs/05-fable-handoff.md를 먼저 읽어라.
docs/02-game-design.md의 게임 "PULSE"를 HTML5(Vite + TypeScript + Canvas)로 구현한다.

성공 기준(모두 충족될 때까지 루프):
1. npm run dev 로 로컬 실행되고 첫 입력에 반응한다.
2. 코어 루프 동작: 시작 → 플레이(정확 히트 시 점수·콤보·속도↑) → 게임오버 → 광고 부활 → 이어하기.
3. 이펙트: 히트/실패/콤보에 파티클·화면흔들림·플래시.
4. iPhone 햅틱: docs/03의 어댑터 전략대로 (성공/실패/콤보). 미지원 시 no-op 폴백.
5. 광고: src/platform 어댑터 뒤로 추상화. 웹은 폴백(카운트다운), TikTok은 SDK 자리 확보.
6. npm run build 성공, dist < 50MB, 60fps.

먼저 플레이 가능한 최소 프로토타입을 만들고, 손맛(juice)이 좋아질 때까지 반복 개선하라.
완성되면 웹으로 배포해 iPhone Safari에서 플레이되는 링크를 만들어라.
```

## 4. 절대 원칙 (CLAUDE.md 요약 — 어기지 말 것)

1. **Think Before Coding** — 가정은 말하고, 불확실하면 물어라.
2. **Simplicity First** — 최소 코드. 추측성 기능·과한 추상화 금지. 게임도 코드도 "깔끔·심플".
3. **Surgical Changes** — 건드릴 것만. 무관한 리팩터링 금지.
4. **Goal-Driven** — 위 성공 기준이 충족될 때까지 루프하고, 스스로 검증하라.

## 5. 고정 제약 (변경하려면 먼저 질문)

- 결과물은 **HTML5**, 패키지 **< 50MB**, 목표 **60fps**.
- 플랫폼 기능(광고·햅틱·저장)은 **어댑터 계층** 뒤로. 게임 로직에서 SDK 직접 호출 금지.
- 게임은 **무한**하고, **원탭**이며, **광고 부활로 계속** 플레이 가능해야 한다.
- iPhone 햅틱은 **향상**이지 필수 의존이 아니다(없어도 완전 동작).

## 6. 배포 준비물 (Fable이 알아야 할 것)

- 웹 데모: 이 환경에 **Netlify / Vercel** MCP 커넥터가 있어 세션 내 배포 가능. (또는 GitHub Pages)
- TikTok 제출: `developers.tiktok.com` 계정·앱 등록 필요(사용자 액션). 상세는 `docs/04`.

## 7. 첫 마일스톤 제안 (Fable이 잡을 순서)

1. **M1 — 플레이 가능한 코어**: 바늘 회전 + 탭 판정 + 점수/게임오버. (손맛 없어도 됨)
2. **M2 — Juice**: 이펙트·햅틱·사운드로 "딱" 맞는 손맛.
3. **M3 — 무한 루프 완성**: 광고 부활/이어하기 + 결과·공유 화면.
4. **M4 — 배포**: 웹 빌드 → 배포 → iPhone 실기 검증 → 밸런싱 튜닝.
5. **M5 — TikTok**: SDK 래핑 → 인앱 프리뷰 → 제출.

각 마일스톤 끝에서 `CLAUDE.md`의 검증 게이트를 통과했는지 스스로 확인하라.
