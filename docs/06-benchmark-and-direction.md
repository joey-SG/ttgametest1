# 06 · 벤치마크 & 기획 방향 (living document)

> **이 문서의 목적**: 개선을 거듭해도 프로젝트가 방향과 목적을 잊지 않도록,
> 키 컨셉·벤치마크·의사결정을 한곳에 기록한다.
> 밸런스·기능 결정을 바꿀 때는 반드시 이 문서의 해당 절과 **의사결정 로그**를 갱신한다.

## 1. 변하지 않는 키 컨셉 (건드리려면 먼저 질문)

1. **원탭 정밀 타이밍** — 조작은 탭 하나. 3초 안에 이해된다.
2. **무한 런 + 0초 재시작** — 엔딩 없음. 죽음은 처벌이 아니라 재도전 초대.
3. **초록=생존, 노랑=보상** — 일반 히트는 런을 절대 끊지 않고, 퍼펙트만이 배율을 올린다.
4. **깔끔·심플 + 과감한 juice** — 미니멀 네온, 이펙트·햅틱은 "딱" 맞는 순간에 몰아준다.
5. **광고는 강요가 아니라 선택** — 부활/2배 모두 유저가 "더 하고 싶어서" 보는 구조.

## 2. 벤치마크 맵 — 무엇을 어디서 가져왔나

전략: **잘되는 게임을 거의 흡사하게 가져오되, 각 게임에서 검증된 조각만 조합**한다.
새 발명은 조합이 검증된 뒤에만.

| 벤치마크 | 가져온 것 (거의 그대로) | 우리가 다르게 한 것 |
|---|---|---|
| **Pop the Lock** | 코어 판정: 회전 침 + 존 탭, 지나침=실패, 히트마다 방향 반전·가속 | 존을 2단(초록 생존/노랑 퍼펙트)으로 분리 |
| **Beatstar** | 퍼펙트 스트릭 → 점수 배율(×2/×3/×4), 배율이 깨져도 런은 계속 | 스트릭 단계 3/6/10 (짧은 런에 맞게 압축) |
| **Knife Hit** | 런에서 재화 획득 → 코스메틱+α 무기 메타 (수집·성장 리텐션) | 무기 보너스를 "히트당 점수"로 (HP 시스템 없음 유지) |
| **Subway Surfers** | 부활은 **런당 1회** 원칙 (점수 가치·긴장감 보존) | 부활을 코인이 아닌 보상형 광고로 |
| **Knife Hit (보스 스테이지)** | 주기적 스테이지 마일스톤 + 진입 보상 = "다음 보스까지" 근접 목표 | 보스 대신 콤보 10/25/50/100/200에서 닌자 색 등급 상승 (docs/07 R2) |

### 부활·2배 규칙 (확정)

- **광고 부활: 런당 1회.** 광고를 스킵하면 횟수를 소모하지 않는다.
  무한 부활은 "죽음이 아깝다"는 감정을 파괴하고 점수의 의미를 없앤다. 1회 부활은
  "가장 아까운 죽음 한 번을 구제"하는 장치로만 쓴다.
- **점수 2배: 런당 1회.** (게임오버 화면에서, 사용 후 버튼 숨김)
- 규칙 변경(예: 부활 2회째는 광고 2개 시청)은 지표 확인 후에만 검토.

## 3. 난이도 철학

- 목표 곡선: **초반 완만 → 중반 가파름 → 천장 유지**.
  초심자는 콤보 ~15까지 "손에 익는 재미"를 느껴야 하고, 상급자의 스킬 천장은 낮추지 않는다.
- 런이 길어지면 죽음이 아까워져 **부활 광고 가치가 오히려 상승**한다.
  단, 게임오버 화면 노출 빈도(광고·상점 접점)를 죽이지 않는 선에서.
- **1차 튜닝 적용됨 (2026-07-04, 사용자 위임)**: `speedRamp 22→28`, `arcShrinkRamp 30→38`, `maxSpeed 유지`.
  제안값(30/40)보다 보수적으로 적용 — 지표(퍼펙트 비율·평균 런 길이)로 재검증 후 조정.

## 4. 상점(무기고) 접점

- **게임오버 직후**(골드가 방금 입금된 순간)와 **타이틀** 두 곳. 플레이 중 진입은 금지(몰입 파괴).

## 5. 측정 (밸런스 결정의 근거) — ✅ 구현됨

로컬 집계 (서버 불필요, 이 기기에만 저장. `pulse.stats` KV, 타이틀 → 📊 지표):

- 평균 콤보 / 런 길이(초) / 평균 점수 / 세션당 재시도 횟수 / 부활 광고 수락률 / 퍼펙트 비율(목표 60~80%).
- 집계 규칙: 런 기록은 게임오버 시 스테이징되고, **부활하면 폐기**(런이 이어지므로),
  다음 런 시작 시 확정된다. 부활 노출/수락은 게임오버·부활 시점에 즉시 기록.
- **사용법**: 며칠 플레이 후 지표를 보고 §3의 난이도 제안값 적용 여부를 결정한다.
  퍼펙트 비율이 60% 미만이면 퍼펙트 존이 너무 좁은 것, 80% 초과면 너무 넓은 것.
