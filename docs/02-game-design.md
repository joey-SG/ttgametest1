# 02 · 게임 디자인 스펙

리서치 결론(`01-market-research.md`)을 하나의 구체적 게임으로 확정한다.
Fable은 이 스펙을 "무엇을 만족해야 하는가"로 읽고, 구현 방식은 자유롭게 선택한다(Goal-Driven).

---

## 1. 게임 컨셉 (확정)

### 이름(가칭): **PULSE**

**한 문장**: 화면 중앙에서 회전하는 바늘을 **정확한 타이밍에 탭**하여 링 위의 타겟을 맞히는
원탭 정밀 리듬 아케이드. 맞힐수록 빨라지고, 빗나가면 끝. 광고를 보면 부활해 콤보를 이어간다.

**왜 이 컨셉인가 (설계 근거):**
- **무한**: 난이도(속도)만 오르는 endless 구조. 엔딩 없음.
- **원탭·심플**: 조작은 탭 하나. 3초 안에 이해.
- **juice 친화적**: 정확히 맞히는 "딱" 순간에 파티클·플래시·스케일 펀치·햅틱을 몰아줄 수 있음.
- **햅틱 자연스러움**: 성공(정확)/실패(충돌)/콤보 마일스톤에 명확한 촉각 이벤트 매핑.
- **바이럴**: "몇 콤보까지 갔나" 점수 경쟁 + 근접 실패 클립이 잘 나옴.

### 대안 컨셉 (Fable 판단으로 교체 가능, 동일 요구사항 만족 시)

- **STACK-FLOW**: 위에서 흐르는 블록을 탭으로 정확히 쌓기(정밀 스택). 완벽 정렬 시 콤보·햅틱.
- **DODGE-LINE**: 한 손가락으로 좌우 라인 스위칭, 끝없이 다가오는 장애물 회피.

> 세 컨셉 모두 "무한 + 원탭 + juice + 햅틱 + 광고 부활"을 만족한다.
> Fable은 프로토타입에서 가장 손맛(juice) 좋은 것을 골라도 된다. 기본 채택은 **PULSE**.

## 2. 코어 루프 (반드시 구현)

```
[시작 화면: 탭하여 시작]
        │  탭
        ▼
[플레이] ── 정확 히트 → 점수+, 콤보+, 속도↑, 이펙트+햅틱(성공)
        │
        └─ 빗나감/충돌 → 이펙트+햅틱(실패)
                          ▼
                   [게임 오버 화면]
                    ├─ "다시하기" (0초 재시작)
                    ├─ "광고 보고 부활" → 보상형 광고 → 콤보 유지하고 이어하기
                    └─ "점수 공유"
```

**불변 규칙:**
- 재시작은 **0초**. 로딩·모달 대기 없음.
- 게임오버는 처벌이 아니라 **재도전 초대**로 연출.

---

## 3. 수익화 & 무한 지속 (보상형 광고)

TikTok Mini Games의 **Rewarded Video Ads**(15~30초, 완주 시 보상)를 코어 루프에 내장:

- **Watch to Revive (부활)**: 게임오버 시 광고 완주 → 마지막 지점에서 콤보 유지하고 부활.
- **Double Score (2배 보상)**: 세션 종료 시 광고 완주 → 최종 점수 2배.
- (선택) **Continue Streak**: 데일리 연속 플레이 보너스.

핵심: **부활 = 무한 플레이의 엔진**. 유저가 광고를 "강요"가 아니라 "더 하고 싶어서" 보게 만든다.

> 광고 SDK 호출은 어댑터 뒤로 추상화한다(`docs/03-tech-architecture.md`).
> 광고 콜백은 boolean(완주 여부)을 반환 — 완주했을 때만 보상 지급.

## 4. 아트 디렉션 — "깔끔하고 심플"

- **미니멀 네온/플랫**: 단색 배경 + 고대비 포인트 컬러 1~2개. 텍스트 최소.
- 절차적/벡터 그래픽 우선(에셋 용량 최소화, 50MB 제한 대응).
- 폰트 1개, UI 요소 최소. "한 화면에 이해되는" 구성.
- 다크 모드 기본(피드 안에서 눈에 띄고 이펙트가 살아남).

## 4.5 판정·배율 규칙 (확정 — 2026-07 리서치 기반)

유사 게임 조사 결과(Beatstar 스트릭 배율 모델, 하이퍼캐주얼 타이밍 설계 가이드) 채택한 규칙:

- **초록(일반 히트)** = 생존. 콤보·속도 상승 유지. 런은 절대 안 끊김.
- **노랑(PERFECT)** = 보상. 연속 퍼펙트 스트릭 3/6/10 도달 시 점수 배율 ×2/×3/×4.
- 일반 히트는 **배율만 ×1로 리셋**하고 런·콤보는 유지 (퍼펙트만 콤보 인정 ❌ — 초심자 이탈 유발).
- 죽음은 오직 빗나감/지나침. 부활 시 콤보 유지, 배율은 리셋.
- 근거: 배율은 "완벽 추구" 반복 동기(리텐션)를 만들고, 일반 히트 생존 보장은
  진입 장벽을 낮춘다. 퍼펙트 존은 도전적이어야 함(비퍼펙트가 20~40% 나오는 폭).

## 5. 이펙트(juice) 체크리스트

- [ ] 정확 히트: 스케일 펀치 + 파티클 버스트 + 짧은 화면 플래시.
- [ ] 콤보 상승: 색 변화 / 트레일 / 사운드 피치 상승.
- [ ] 실패: 화면 흔들림(shake) + 크랙/디졸브 파티클.
- [ ] 콤보 마일스톤(10/25/50…): 특별 이펙트 + 강한 햅틱.
- [ ] 모든 이펙트는 60fps 유지 선에서. 과하면 "심플" 원칙 위배.

## 6. 사운드

- 짧고 만족스러운 SFX(히트/실패/콤보). 절차적 생성 또는 초경량 파일.
- 음소거 상태(TikTok 피드는 무음 시청 많음)에서도 **햅틱+비주얼만으로** 손맛이 완성돼야 함.

## 7. 화면 목록 (최소)

1. **타이틀/시작** — 로고 + "탭하여 시작" + 최고점수.
2. **플레이** — 게임 + 현재 점수/콤보(작게).
3. **게임오버** — 점수, 최고점수 갱신 강조, [다시][광고 부활][공유].

> 그 외 설정·상점 등은 MVP 범위 밖. Simplicity First.
