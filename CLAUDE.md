> 이 파일은 Andrej Karpathy가 정리한 LLM 코딩 실패 모드 관찰에서 파생된
> 4대 원칙(multica-ai/andrej-karpathy-skills)을 **이 TikTok 게임 프로젝트에 맞게
> 구체적 성공 기준으로 재작성한 것**이다. 원본을 템플릿이 아니라 "메뉴"로 사용한다.

---

## 프로젝트 한 줄 정의

TikTok에서 바이럴 가능한 **무한(endless) 원탭 아케이드 게임**.
깔끔·심플한 조작 + 만족스러운 이펙트 + iPhone 햅틱 + 보상형 광고로 "한 번 더" 무한 재도전.
빌드 결과물은 **HTML5**이며, 웹(Netlify/Vercel) 데모와 TikTok Mini Game 제출 양쪽으로 배포한다.

---

## 1. Think Before Coding (먼저 생각하라)

- 가정은 **명시적으로 말하고**, 불확실하면 코딩 전에 질문한다. 조용히 추정하고 진행하지 않는다.
- 여러 해석이 가능하면 트레이드오프를 나열한 뒤 하나를 고른 이유를 밝힌다.
- **성공 기준**: 코드를 쓰기 전에 "무엇이 완료의 정의인가"를 한 문장으로 말할 수 있다.

## 2. Simplicity First (단순함이 먼저)

- 문제를 푸는 **최소 코드**. 추측성 기능·추상화·일어나지 않을 예외 처리는 넣지 않는다.
- 게임은 "깔끔하고 심플한 작동"이 제품 요구사항이다. 코드도 동일 원칙을 따른다.
- **성공 기준**: 새 의존성/추상화 계층을 추가할 때마다 "이게 없으면 지금 무엇이 안 되는가"에 답할 수 있다. 답이 없으면 넣지 않는다.

## 3. Surgical Changes (외과적 변경)

- 건드려야 하는 것만 건드린다. 요청받지 않은 리팩터링·인접 코드 "개선"을 하지 않는다.
- 기존 스타일 컨벤션(네이밍, 포맷)을 그대로 따른다.
- **성공 기준**: `git diff`가 요청한 변경과 직접 관련된 라인만 담는다. 무관한 diff 노이즈가 없다.

## 4. Goal-Driven Execution (목표 주도 실행)

- 단계별 지시가 아니라 **검증 가능한 성공 기준**을 정의하고, 충족될 때까지 루프한다.
- **이 프로젝트의 검증 게이트 (변경 후 반드시 통과):**
  - `npm run build` 성공, 빌드 산출물 총합 **< 50MB** (TikTok Mini Games 제한).
  - 게임이 로컬(`npm run dev`)에서 로딩되고 첫 입력에 반응한다.
  - 코어 루프가 동작한다: 시작 → 플레이 → 게임오버 → (광고 보상) 재도전 → 점수 갱신.
  - `console.error`가 정상 플레이 경로에서 발생하지 않는다.
  - 60fps 목표: 저사양 모바일에서 프레임 드랍이 눈에 띄지 않는다.

## 게임 제품 요구사항 (이 프로젝트 고유의 성공 기준)

이 항목들은 "완료의 정의"다. 구현이 아래를 만족하지 못하면 미완료다.

- [ ] **무한 장르**: 끝(엔딩)이 없다. 점수/거리/콤보를 무한히 쌓는 구조.
- [ ] **원탭/심플 조작**: 튜토리얼 없이 3초 안에 조작을 이해할 수 있다.
- [ ] **이펙트(juice)**: 성공/실패/콤보에 파티클·화면 흔들림·색 변화 등 시각 피드백이 있다.
- [ ] **iPhone 햅틱**: 핵심 순간(성공/충돌/콤보)에 햅틱이 발생한다. (구현 상세: `docs/03-tech-architecture.md`)
- [ ] **광고 보상 재플레이**: 게임오버 시 "광고 보고 이어하기/부활" 옵션으로 무한 지속 가능.
- [ ] **바이럴성**: 점수 공유/스크린샷/짧은 클립을 유도하는 결과 화면.

## 플랫폼/스택 고정값 (임의 변경 금지 — 변경 시 질문)

- 결과물 포맷: **HTML5** (Canvas 기반). 무거운 3D 엔진 금지.
- 플랫폼 기능(광고·햅틱·로그인)은 **어댑터 계층 뒤로 추상화**한다.
  웹 데모에서는 mock/폴백, TikTok 인앱에서는 `TTMinis.game` SDK로 스왑.
- 패키지 크기: **< 50MB**. 에셋은 압축/절차적 생성 우선.
